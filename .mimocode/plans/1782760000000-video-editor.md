# Video Editor Implementation Plan

## Overview
Build a moderate-featured video editor as a Lightbox overlay. Multi-track timeline, trim/split, text overlays, audio, transitions, effects, speed control, subtitles (manual + AI-generated), and FFmpeg export. HTML5 video + canvas overlay for preview.

---

## Architecture

### Entry Point
- Opens as a full-screen overlay from Lightbox (like existing `EditingMode` for images)
- Triggered via "Edit" button on video items in Lightbox
- Loads current video + allows adding more from library

### Preview Approach
- `<video>` element for base playback
- `<canvas>` overlay for text/effects rendered at current playback time
- Synchronized: canvas redraws on `timeupdate` event

### Export Pipeline
- Frontend sends project JSON (timeline, clips, effects, text) to backend
- Backend renders via FFmpeg: concatenation, filters, text overlays, audio mixing
- Returns rendered video file

## Phase 1: Core Editor Shell & State

### Step 1.1: Create Zustand store
**New file:** `frontend/store/videoEditorStore.ts`

```typescript
interface VideoEditorProject {
  id: string;
  name: string;
  tracks: Track[];
  duration: number; // total timeline duration in seconds
  currentTime: number;
  isPlaying: boolean;
  zoom: number; // pixels per second
}

interface Track {
  id: string;
  type: 'video' | 'audio' | 'text' | 'subtitle';
  name: string;
  clips: Clip[];
  muted: boolean;
  volume: number; // 0-1
}

interface Clip {
  id: string;
  type: 'video' | 'audio' | 'text' | 'subtitle';
  sourcePath?: string; // for video/audio
  startTime: number; // position on timeline (seconds)
  duration: number; // clip duration on timeline
  trimStart: number; // trim offset from source start
  trimEnd: number; // trim offset from source end
  speed: number; // 0.25 - 4.0
  // Text-specific
  text?: string;
  fontFamily?: string;
  fontSize?: number;
  fontColor?: string;
  fontWeight?: string;
  textAlign?: string;
  x?: number;
  y?: number;
  // Transition
  transitionIn?: TransitionType;
  transitionOut?: TransitionType;
  // Effects
  effects?: Effect[];
}

type TransitionType = 'none' | 'crossfade' | 'fadeBlack' | 'fadeWhite';

interface Effect {
  type: string; // matches filterEngine Adjustments keys
  params: Record<string, number>;
}
```

Store actions: `setPlaying`, `setCurrentTime`, `addTrack`, `removeTrack`, `addClip`, `removeClip`, `updateClip`, `splitClip`, `trimClip`, `reorderClips`, `setZoom`, `addTextOverlay`, `updateTextOverlay`.

## Phase 6: Backend Export API

### Step 6.1: Export endpoint
**New file:** `backend/app/api/video/__init__.py`
**New file:** `backend/app/api/video/export.py`

```
POST /api/v1/video/export
Body: {
  tracks: [...],  // timeline data
  resolution: [1920, 1080],
  fps: 30,
  format: "mp4"
}
Response: { "job_id": "...", "status": "processing" }
```

### Step 6.2: Export rendering service
**New file:** `backend/app/services/video_export.py`

FFmpeg command builder:
1. Input: all source video/audio files
2. Filter complex: trim, speed, concat, text overlays (drawtext), audio mix, transitions
3. Output: H.264 MP4 with AAC audio

Key FFmpeg filters:
- `trim=start=X:end=Y` — clip trimming
- `setpts=PTS*speed` — speed change
- `drawtext=text=...:fontsize=...:fontcolor=...` — text overlays
- `amix` — audio mixing
- `xfade=transition=fade:duration=0.5` — transitions
- `concat` — joining clips

### Step 6.3: Export status endpoint
```
GET /api/v1/video/export/{job_id}
Response: { "status": "processing" | "completed" | "failed", "progress": 0.5, "output_path": "..." }
```

### Step 6.4: Download endpoint
```
GET /api/v1/video/export/{job_id}/download
Response: Video file (FileResponse)
```

## Phase 7: AI Subtitle Generation

### Step 7.1: Add feature flag
**Modify:** `backend/app/config.py`

```python
ENABLE_AI_SUBTITLES: bool = False
```

### Step 7.2: Subtitle generation service
**New file:** `backend/app/services/subtitle_gen.py`

Pipeline:
1. Extract audio from video via ffmpeg: `ffmpeg -i video.mp4 -vn -acodec pcm_s16le audio.wav`
2. Run faster-whisper for transcription with word-level timestamps
3. Run wav2vec2 alignment for precise word-level timing
4. Group words into subtitle segments (max 42 chars per line, max 2 lines)
5. Return SRT-format subtitle data

Dependencies (optional):
- `faster-whisper` — CTranslate2-based Whisper implementation
- `wav2vec2` — Facebook's speech recognition model for alignment
- Both are opt-in, only loaded when `ENABLE_AI_SUBTITLES=True`

### Step 7.3: Subtitle generation endpoint
**New file:** `backend/app/api/video/subtitles.py`

```
POST /api/v1/video/subtitles/generate
Body: { "photo_id": 123 }
Response: { "subtitles": [{ "start": 0.5, "end": 2.1, "text": "Hello world" }, ...] }
```
